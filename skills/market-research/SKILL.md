# 市场规模调研报告技能

本技能指导生成专业级市场规模调研报告，涵盖完整的TAM/SAM/SOM框架、多维度数据收集与交叉验证、可视化图表，最终输出结构化HTML报告。

## 工作流程总览

```
1. 理解需求 → 2. 构建框架 → 3. 数据调研(3路并行) → 4. 交叉验证 → 5. 生成报告
```

---

## Step 1：需求解析

收集以下关键信息（如用户未提供，需合理推断或询问）：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 目标行业/产品 | 具体细分领域，越精确越好 | 必填 |
| 地理范围 | 全球 / 特定区域 / 单国 | 全球 |
| 基准年 | 数据起始年份 | 当前年 |
| 预测期 | 3年或5年 | 5年 |
| 竞争切入视角 | 初创公司 / 成熟企业 / 投资分析 | 通用 |

## Step 2：TAM / SAM / SOM 三层框架

### 概念定义
- **TAM（Total Addressable Market）**：总可寻址市场 — 假设100%市场份额时的理论最大营收
- **SAM（Serviceable Addressable Market）**：可服务市场 — 基于地理、能力、法规等实际约束筛选后的市场
- **SOM（Serviceable Obtainable Market）**：实际可获市场 — 在竞争格局下3-5年内可实际占领的份额

### 计算规则
```
TAM = 潜在用户总数 × 平均客单价（全球/行业宽口径）
SAM = TAM × 地理渗透率 × 产品适配度 × 法规可及性
SOM = SAM × 市场份额预期 × 执行折扣系数（通常0.1~0.3）
```

### 验证关系
- SAM/TAM 通常在 10%~40% 之间
- SOM/SAM 通常在 5%~20% 之间（初创）或 15%~40%（成熟企业）
- 如比例异常，必须在报告中说明原因

## Step 3：三种数据收集方法（并行执行）

### 方法A：自上而下（行业报告法）
**数据来源优先级：**
1. **Tier 1权威机构**：Gartner、IDC、Forrester、McKinsey Global Institute、BCG、Bain
2. **Tier 2专业机构**：Grand View Research、MarketsandMarkets、Mordor Intelligence、Statista
3. **Tier 3补充来源**：政府统计局、行业协会、上市公司年报（10-K / 20-F）、央行报告

**搜索策略：**
```
查询模板："{行业} market size {年份} {Gartner|IDC|Forrester|McKinsey}"
交叉查询："{行业} CAGR forecast {年份}-{年份+5}"
补充查询："{行业} total addressable market report {当前年}"
```

使用 `web_search` 工具执行至少 **6次不同查询**，覆盖不同机构和年份。

### 方法B：自下而上（客户分层法）
**执行步骤：**
1. 识别目标客户群体（企业/消费者/政府）
2. 按规模/行业/地区分层
3. 估算各层客户数量（来源：行业报告、人口统计、商业数据库）
4. 乘以各层平均客单价
5. 汇总得到底层市场规模

**公式：**
```
底层TAM = Σ（细分客户数 × 细分平均价格）
```

### 方法C：价值理论法（支付意愿）
**执行步骤：**
1. 识别客户核心痛点/价值获取
2. 量化替代方案成本（现有解决方案的支出）
3. 基于价值占比估算合理定价
4. 推算市场规模

**公式：**
```
价值定价 = 客户节省成本 × 价值捕获率（通常10%~30%）
市场规模 = 目标客户数 × 价值定价
```

## Step 4：数据交叉验证

### 验证矩阵
| 维度 | 自上而下 | 自下而上 | 价值理论 | 偏差判定 |
|------|----------|----------|----------|----------|
| TAM | $X₁B | $X₂B | $X₃B | |
| CAGR | Y₁% | Y₂% | Y₃% | |

### 偏差处理规则
- **偏差 < 20%**：取加权平均（A:40% + B:40% + C:20%）
- **偏差 20%~50%**：分析差异原因，以Tier 1机构数据为主锚，说明调整逻辑
- **偏差 > 50%**：标注数据不确定性，提供区间估算，建议用户补充一手调研

### CAGR计算
```
CAGR(n年) = (终值/初值)^(1/n) - 1

3年CAGR = (Year+3 / BaseYear)^(1/3) - 1
5年CAGR = (Year+5 / BaseYear)^(1/5) - 1
```

## Step 5：HTML报告生成

### 报告结构（必须包含所有章节）

```html
<!-- 章节顺序 -->
1. 报告封面（标题、行业、日期、摘要KPI卡片）
2. 执行摘要（核心数字：TAM/SAM/SOM + CAGR，<200字）
3. 市场定义与范围（地理、产品、客户边界）
4. TAM分析（三种方法 + 交叉验证结果）
5. SAM分析（细分逻辑 + 筛选条件）
6. SOM分析（竞争格局 + 份额预测）
7. 增长驱动因素（3-5个关键驱动）
8. 风险与制约因素（2-3个关键风险）
9. 数据来源与方法论说明
10. 可视化图表区域（见下方图表规格）
```

### 可视化图表规格（使用 Chart.js via CDN）

**必须包含的图表：**

1. **TAM/SAM/SOM 同心圆/漏斗图**（SVG手绘或Chart.js Doughnut）
2. **市场规模增长折线图**（BaseYear → +5年，含3条线：TAM/SAM/SOM）
3. **三方法交叉验证柱状图**（对比A/B/C三种方法的估算值）
4. **CAGR对比条形图**（3年 vs 5年，与行业基准对比）
5. **增长驱动因素雷达图**（可选，如驱动因素≥4个）

## 注意事项

1. **数据诚实性**：若某机构数据无法验证，必须注明"估算"或"基于公开信息推算"
2. **区间优于点估计**：对不确定性较高的数据，给出低/基/高三种情景
3. **中英文数字格式**：中文报告用"亿元/百亿美元"，英文报告用"$XB/$XM"
4. **避免过度精确**：市场规模精确到亿级即可，不需要精确到百万位
5. **行业特殊性**：新兴行业（AI、量子计算等）需特别说明预测不确定性
